# 混合场景资源生命周期

> 状态：**Draft for review**。定义生命周期责任，未实现任何资源管理代码。

## 1. 生命周期状态

| 状态 | 责任 | 可观察证据 |
|---|---|---|
| 初始化 | 创建统一渲染上下文、图层和取消域 | 单一 Renderer/Loop、状态日志 |
| 加载 | 并行但分层请求 baseLayers 与 devices | 请求、加载状态、资源归属 |
| 更新 | 统一循环更新 Tiles、GLB 动画与标注 | FPS、错误、动画证据 |
| 隐藏 | 停止可见性但保留经批准缓存 | 内存/对象计数 |
| 卸载 | 解除场景图、对象注册和事件 | 引用清单、候选拾取清理 |
| 销毁 | 取消请求、停止循环、释放 WebGL 资源 | 无重复循环、内存趋势 |

## 2. 场景切换与错误恢复

- 场景切换须先取消上一场景未完成请求，再按资源所有者卸载；不得让陈旧响应写入新场景。
- 缓存键、上限、失效策略和跨场景复用规则均为 TBD - 待 POC 或现场资料确认。
- Tiles 加载、解析或更新失败时，隔离 TilesLayer、保留 GLB DeviceLayer，并暴露可识别 fallback 状态。
- GLB 加载失败时按既有 GLB fallback 工作，不把失败误报成 Tiles 成功。

## 3. 释放清单

页面退出、场景切换或明确关闭图层时必须按实际资源类型检查：

1. animation frame、订阅、DOM/输入事件和计时器。
2. 中止中的网络请求、解析任务和动态 tile 内容引用。
3. scene graph 节点、Raycaster 候选注册、对象树缓存。
4. geometry、material、texture、render target、renderer 及 Tiles renderer 持有资源。
5. 错误状态、重试队列和调试辅助对象。

POC 必须用重复进入/退出与失败注入记录证据，不能以一次页面关闭替代生命周期验证。
