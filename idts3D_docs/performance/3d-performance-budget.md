# 3D 性能预算与 POC 暂定基线

> 状态：Draft for review。以下数值仅为 POC 可执行的暂定基线，不是生产阈值、现场承诺或已实测结果。

## 1. 固定测试环境

| 项目 | 暂定要求 |
|---|---|
| 浏览器 | 当前稳定版 Chrome 或 Edge；记录完整版本 |
| 分辨率 | 1920 × 1080 |
| 页面缩放 | 100% |
| 硬件 | 记录实际 CPU、GPU、内存与 GPU 驱动版本 |
| 网络 | 本地静态服务，或记录的固定测试网络 |
| 缓存 | 冷缓存 3 次、暖缓存 3 次 |
| 观察窗口 | 每轮稳定后 60 秒 |

## 2. 阶段 A 暂定门槛

- 平均 FPS 不低于 45，1% Low FPS 不低于 25。
- 可操作时间中位数不高于 8 秒。
- 连续进入/退出 10 次后功能正常。
- 稳定内存相对第一次稳定值增长不高于 20%，且绝对增长不高于 200 MB。
- 0 个未处理异常、0 个重复 Animation Loop、0 个重复关键事件监听。
- Tiles 失败时 GLB fallback 必须通过。

## 3. 阶段 B 暂定门槛

- 平均 FPS 不低于 30，1% Low FPS 不低于 18。
- 可操作时间中位数不高于 20 秒。
- 相对纯 GLB 对照组的平均 FPS 下降不超过 40%。
- 连续进入/退出 10 次后无持续线性内存增长。
- Tiles 失败时 GLB fallback 必须通过。

## 4. 坐标与测量方法

阶段 A 的三个标定点相对误差不高于参考跨距的 1%。阶段 B 必须通过显式变换稳定复现；现场绝对容差未确认时只能 Conditional Go，无法稳定复现或每次需人工试调则 No-Go。

必须记录：FPS 采样工具与 60 秒窗口；首屏和可操作时间起止点；冷/暖缓存轮次；纯 GLB 对照组；内存稳定等待时间；失败注入方式；进入/退出次数；原始证据保存位置。没有实际数据时填写 TBD。
