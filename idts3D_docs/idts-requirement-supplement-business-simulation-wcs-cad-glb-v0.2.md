## 1. 本次新增需求总述

本次补充不只是“模型展示”，而是将 IDTS 数字孪生系统进一步明确为具备以下能力的正式业务系统：

1. 支持通过项目 CAD 资料进行场景建模，渲染 CAD 规划中的建筑物、厂房、楼栋、楼层、区域、货架、地堆、输送线等静态结构。
2. 支持渲染 GLB 模型，用于展示提升机、堆垛机、AGV、输送线关键段、机械手等可交互或可动业务设备。
3. 支持模型仿真，包括设备运动仿真、任务流程仿真、路线仿真、业务分段任务仿真、产能仿真和运行效率分析展示。
4. 支持仿真与 WCS 系统实时对接，通过 WCS 或上游系统推送的实时数据驱动数字孪生模型变化。
5. 支持与 WMS、MES、ERP 等上游系统进行数据交互，显示订单、库存、任务、设备状态、故障、效率、报表等信息。
6. 支持电视大屏展示、触摸或鼠标交互、视角切换、缩放旋转、物资查询、告警管理、虚拟摄像头和多客户端展示。

本次补充需要进入后续 Word 方案，但不代表全部进入 MVP 一期开发范围。正式开发时应区分：

- MVP 一期：模型资产、GLB 上传、manifest、object-tree、可动部件配置、motion target、场景设备绑定、前端加载设备 GLB。
- 后续阶段：模型仿真、WCS 实时对接、AGV/堆垛机/提升机/输送线业务仿真、报表、权限、远程控制、完整 CAD/3D Tiles 生产化。

## 2. 与现有总体技术方案的关系

现有 v1.2 技术方案已经确定：

- 前端采用 Vue 3 + TypeScript + Three.js。
- 后端固定采用 C# / ASP.NET Core Web API / .NET 8 / EF Core 8。
- 厂区、建筑、楼层、地面、固定结构采用 3D Tiles 作为静态大场景底座。
- 业务设备采用独立 GLB，用于可动部件、任务动画、异常高亮和设备交互。
- 后端缓存派生模型资产，不缓存浏览器 GPU 渲染状态。

本次需求补充在该路线基础上增加业务能力：

```text
CAD 规划建筑物 / 区域建模 → 3D Tiles 或静态 GLB 底座
GLB 设备模型 → 提升机 / 堆垛机 / AGV / 输送线 / 机械手
WCS 实时数据 → 驱动设备状态、位置、动作、任务、异常
仿真模块 → 根据任务、路径、设备状态和历史数据进行可视化仿真或分析展示
业务系统对接 → WMS / MES / ERP / WCS 数据接入与展示
```

## 3. 场景描述需求整理

### 3.1 场景建模

需求：

- 按项目 CAD 所涉及区域进行建模和渲染。
- 支持 CAD 规划中的建筑物、厂房、楼栋、楼层、区域、地面、货架、库位、地堆、输送线和设备布局的三维呈现。
- 场景建模应覆盖项目实际范围，并保证空间比例、设备位置、路径关系和货位关系与现场规划一致。

建议实现边界：

- CAD 规划建筑物和厂区静态结构优先转换为 3D Tiles 或静态 GLB。
- 需要业务控制的设备仍以独立 GLB 方式接入。
- 场景模型应具有版本管理，支持发布、回滚和现场重新校准。

### 3.2 AGV 路线展示

需求：

- 按照实时坐标，以三维视角展示 AGV 的运动轨迹。
- 展示 AGV 取货过程、卸货过程和运行过程。
- AGV 运行过程中需要显示和查询当前搬运的物料信息。

展示内容：

- AGV 当前坐标。
- AGV 路线轨迹。
- 当前任务号。
- 起点、终点。
- 当前所搬运物料信息。
- 取货、运输、卸货状态。

### 3.3 AGV 运行展示

需求：

- 在运行过程中实时显示 AGV 当前 X / Y 坐标及运行状态。
- 实时显示 AGV 任务号、电量等信息。
- 点击 AGV 主体可以查看详情。

详情字段：

- 任务号。
- 起点。
- 终点。
- 时间。
- 电量。
- 运行状态。
- 故障详情。

数据来源说明：

- 具体显示信息根据推送的接口定义展示。
- AGV 实时运行数据由甲方或上游系统通过接口提供，乙方负责接入、呈现和提醒。

### 3.4 堆垛机运行展示

需求：

- 三维实时展示堆垛机的运行状态。
- 展示堆垛机行走动作。
- 展示堆垛机取货、放货动作。
- 异常情况需要实时提醒。

展示内容：

- 运行 / 待机 / 故障 / 维护状态。
- 位置坐标。
- 当前任务。
- 货叉动作。
- 取放货动作。
- 当前货物状态。
- 异常提醒和告警详情。

### 3.5 输送线展示

需求：

- 三维实时展示输送线运行状态。
- 展示输送线运行或动作过程。
- 异常情况实时提醒。

展示内容：

- 输送线运行状态。
- 输送线启停状态。
- 当前输送物料信息。
- 堵塞、异常、故障提示。

### 3.6 提升机展示

需求：

- 三维实时展示提升机的运行状态。
- 展示提升机运行和动作过程。
- 异常情况实时提醒。

展示内容：

- 提升机运行 / 待机 / 故障状态。
- 载货台位置。
- 升降高度。
- 当前任务。
- 载货台有货 / 无货。
- 传感器或电机异常提醒。

### 3.7 立体库和地堆展示

需求：

- 三维实时展示货架内的库位状态。
- 显示库位有货、无货等状态。
- 支持查询物料并高亮显示。
- 支持点击物料查看详情信息。

展示内容：

- 货位状态。
- 库位编码。
- 托盘编码。
- 物料编码。
- 物料名称。
- 规格型号。
- 数量。
- 所属订单或入库时间。

## 4. 系统功能需求整理

### 4.1 电视大屏互动

需求：

- 通过高清数据线连接主机与电视机。
- 主机运行数字孪生系统。
- 系统提供一键开关自动视角功能，可在系统配置页面配置。
- 支持在电视大屏上综合展示 AGV、机械手、堆垛机、输送线、立体库的运行情况。

大屏展示内容：

- 库位空满情况。
- 托盘情况。
- 库位状态情况。
- AGV 运行情况。
- 机械手运行情况。
- 堆垛机运行情况。
- 输送线运行情况。
- 立体库运行情况。

交互规则：

- 空料架：点击空料架显示托盘编码。
- 满料架：点击满料架显示物料详细信息。
- 物料详情包括物料编码、物料名称、规格型号、数量等。
- 具体字段根据接口协议展示。

### 4.2 缩放与旋转

需求：

- 支持缩放操作。
- 支持旋转操作。
- 支持不同角度查看场景。
- 如屏幕支持触摸，应支持触摸缩放和旋转。
- 如屏幕不支持触摸，应支持通过鼠标连接主机电脑进行操作。

### 4.3 物资查询

需求：

- 支持按照托盘编码查询。
- 支持按照物料编码查询。
- 支持按照物料名称查询。
- 支持按照规格型号查询。
- 支持模糊匹配定位查询。
- 查询匹配后应对目标进行高亮或特殊标注显示。

### 4.4 警报管理

需求：

- 监控运行中产生的异常记录。
- 记录异常产生时间。
- 记录异常处理情况。
- 支持查看历史告警记录。

告警内容：

- 设备编号。
- 设备类型。
- 故障类型。
- 故障时间。
- 处理状态。
- 处理记录。
- 处理人员，若权限系统提供该数据。

### 4.5 虚拟摄像头

需求：

- 支持自定义虚拟摄像头。
- 虚拟摄像头可实时监控数字孪生画面。
- 虚拟摄像头可用于预设视角、巡检路线、固定大屏视角或设备跟随视角。

## 5. 高精度建模能力需求

### 5.1 模型范围

需支持以下对象的高精度建模：

- 堆垛机立体库。
- 货架。
- 堆垛机。
- 货位。
- 输送线。
- AGV。
- AGV 导航路径。
- AGV 充电区。
- 出入库站台。
- 缓存区。
- 相关配套设备。

### 5.2 建模比例与精度

需求：

- 支持 1:1 等比例建模。
- 模型精度需覆盖设备关键尺寸。
- 设备关键尺寸误差要求：≤ ±5mm。
- 需要覆盖设备运动部件，如堆垛机货叉、AGV 驱动轮、提升机载货台等。
- 需要覆盖物理属性，如材质、颜色、设备编号等。
- 确保数字模型与实物完全匹配。

### 5.3 唯一标识映射

需求：

- 货位、AGV、堆垛机需要唯一标识映射。
- 数字模型需标注设备编号、型号、所属区域、功能属性。
- 模型标识与物理设备的管理编码应一致。

示例：

- “仿真 A 区货架”。
- “AGV-潜伏式-承载 500kg”。
- “堆垛机-01”。
- “提升机-01”。

### 5.4 复杂场景建模

需求：

- 支持多楼层立体库。
- 支持多类型 AGV，如潜伏式、叉车式、牵引式。
- 支持混合调度场景。
- 模型需清晰呈现设备空间布局。
- 模型需清晰呈现路径关联关系，如 AGV 与堆垛机、输送线的接驳点位。

## 6. 实时渲染与交互能力需求

### 6.1 实时渲染

需求：

- 具备高保真实时渲染能力。
- 支持光影模拟，如自然光、设备指示灯。
- 支持材质纹理，如金属货架、塑料货箱。
- 支持动态效果，如堆垛机升降、平移、AGV 行驶。
- 渲染画面帧率要求：≥25fps。
- 操作过程应不卡顿、不拖影。

### 6.2 自定义视角配置

需求：

- 支持全局鸟瞰视图。
- 支持堆垛机特写视图。
- 支持 AGV 路径视图。
- 用户可以一键切换视图。
- 支持视图保存与快速调取。

### 6.3 多模式切换

需求：

- 支持提前预设多种巡检路线。
- 后台配置选择后，自动进入巡检模式。
- 支持设备跟随视角：当运动设备被选定后，视角跟随设备。
- 支持自定义多个区域，首页进行轮播展示实时画面。
- 支持类似游戏的便捷交互方式，包括键盘、鼠标或其他输入设备操作。

### 6.4 平台兼容性与扩展性

需求：

- 兼容 Windows 10 及以上主流桌面操作系统。
- 支持 webapi、websocket 等协议与上游系统对接。
- 在满足性能要求的情况下，也可以协调其他对接协议。

## 7. 堆垛机数字孪生子系统需求

### 7.1 设备状态实时映射

#### 7.1.1 数据采集范围

需要采集堆垛机运行状态，包括：

- 运行。
- 待机。
- 故障。
- 维护。

需要采集位置信息：

- X 轴：货架列。
- Y 轴：货架层。
- Z 轴：升降高度。
- 位置精度：±10mm。

需要采集作业信息：

- 货叉动作：伸 / 缩 / 举升。
- 当前作业任务：取货 / 放货。
- 任务优先级。

需要采集货位状态：

- 空 / 满。
- 存放货物信息。
- SKU。
- 数量。
- 入库时间。
- 所属订单。

需要采集故障信息：

- 故障代码。
- 故障部件，如电机、传感器。
- 故障描述。
- 发生时间。

#### 7.1.2 数据采集频率

- 运行状态、故障信息实时同步，延迟 ≤500ms。
- 位置信息、作业信息采集频率 ≥1 次 / 秒。
- 货位状态随作业更新，作业完成后 1 秒内同步。

#### 7.1.3 异常报警

当堆垛机出现故障，如卡货、电机过载，或货位异常，如货位超重、货物偏移时，系统需立即触发报警。

报警方式包括：

- 模型高亮，红色闪烁。
- 弹窗提示，包含故障详情。
- 客户端声光提醒。

### 7.2 作业流程可视化与仿真

#### 7.2.1 作业可视化

需动态呈现堆垛机的全流程作业，例如：

```text
从入库站台取货
→ 移动至目标货位
→ 放货
→ 返回待机位
```

作业路径需在数字模型中以动态轨迹线标注。

轨迹线示例：

- 绿色表示正常路径。
- 黄色表示临时调整路径或特殊业务路径。

待确认问题：截图中备注提示“这里不应该是临时调整路径，应该是外形检测不合格的从退货出口的路径”。后续 Word 方案应将该处作为待确认业务点，不应直接写死。

#### 7.2.2 路径仿真优化

需求：

- 支持堆垛机作业路径预仿真。
- 可根据任务优先级、设备负载率、路径拥堵情况优化路径。
- 多台堆垛机同时作业时，仿真应避免路径冲突。
- 可输出路径优化前后的效率对比，如作业时间缩短比例。

责任边界：

- 所有数据由甲方通过接口推送方式提供。
- 乙方负责将甲方提供的数据展示出来。
- 乙方不独立分析模拟任何任务，除非双方另行约定合作范围。

#### 7.2.3 产能仿真

需求：

- 支持基于历史数据进行产能仿真，如 1 个月作业量。
- 模拟不同订单量下堆垛机利用率。
- 统计运行时间 / 总时间。
- 统计作业效率，如每小时完成取放货次数。
- 输出产能瓶颈分析，如某台堆垛机负载过高、某区域货位不足。

责任边界同上：甲方提供数据，乙方负责展示和提醒，不独立承担业务分析算法，除非双方约定。

#### 7.2.4 业务仿真模拟

需求：

- 上游 WMS 下发或其他上游系统下发堆垛机、输送线、提升机等设备任务时，系统应支持接收上游系统发出的分段子任务号、起点、终点等数据。
- 分段任务示例：提升机分段任务、输送线分段任务、堆垛机分段任务。
- 在真实设备不动的情况下，模型自主根据输送线路进行仿真运动。
- 每和分段任务对应的设备类型仿真完成后，应上报上游系统完成。
- 业务仿真最终完成后，上报上游系统完成。
- 支持实时模式和业务仿真模式两种模式切换。

责任边界：

- 所有分段任务数据由甲方通过接口推送方式提供。
- 乙方负责呈现和结合分段任务做仿真运动展示。

### 7.3 维护与追溯能力

需求：

- 基于堆垛机运行时长和维护周期生成维护预警。
- 示例：电机维护每 500 小时 1 次。
- 可提前 7 天生成维护预警。
- 可关联维护手册。
- 点击预警可查看维护步骤和所需备件。

责任边界：

- 维护数据由甲方通过接口提供。
- 乙方负责展示和实时提醒。

## 8. AGV 数字孪生子系统需求

### 8.1 动态轨迹与任务管理

#### 8.1.1 实时轨迹映射

需求：

- 采集 AGV 实时位置，坐标精度 ≤ ±50mm。
- 采集频率 ≥2 次 / 秒。
- 采集运行状态：行驶、待机、充电、作业中、故障。
- 采集速度，范围 0~1.5m/s。
- 采集电量百分比，实时显示。
- 数字孪生模型中以动态图标呈现 AGV 各状态图。

#### 8.1.2 任务可视化管理

需求：

- 同步 AGV 任务列表。
- 列表字段包括任务编号、起点、终点、货物信息、优先级、分配时间。
- 在数字孪生系统中可查看任务执行进度，如“已完成 30%”。
- 任务状态用颜色区分。
- 支持按优先级排序。
- 支持任务状态筛选：待执行、执行中、已完成、异常终止。

#### 8.1.3 调度仿真

需求：

- 支持 AGV 调度仿真。
- 模拟任务分配、路径规划、冲突规避过程。
- AGV 在交叉路口相遇时，仿真系统可验证优先级避让策略，如等待队列。
- 当 AGV 空闲率过高或电量不足时，推荐充电调度策略，如调度最近的空闲 AGV 或空闲充电站。
- 可注入 AGV 空闲率等分析数据。

责任边界：

- 所有数据由甲方通过接口推送方式提供。
- 乙方负责将甲方提供的数据展示出来和实时提醒。
- 调度算法、路径规划和模拟任务分析是否由乙方承担，需要单独确认。

### 8.2 充电与故障管理

#### 8.2.1 充电状态映射

需求：

- 采集 AGV 充电状态。
- 状态包括未充电、充电中、充满、充电故障。
- 显示充电电流、电压。
- 显示充电时长。
- 数字模型中需高亮显示充电区。
- 绿色表示空闲。
- 红色表示占用。
- 当 AGV 剩余电量 ≤20% 时，系统自动提示低电量预警。
- 系统可推荐最近的空闲充电位。

#### 8.2.2 故障定位与处理

需求：

- AGV 出现故障时，应立即高亮故障 AGV。
- 故障示例：导航丢失、电池故障、碰撞报警。
- 支持远程排查，通过系统查看 AGV 的传感器数据，如激光雷达扫描结果。
- 辅助判断故障原因。
- 显示故障详情，包括故障类型、可能原因、处理建议。

### 8.3 效率分析与优化

#### 8.3.1 效率统计

需求：

- 按日 / 周 / 月统计 AGV 运行效率。
- 统计行驶时间占比。
- 统计作业时间占比。
- 统计空闲时间占比。
- 统计任务完成率，要求已完成任务数 / 总分配任务数 ≥99.5% 为达标。
- 统计平均作业时长，即单任务从分配到完成的平均时间。
- 通过柱状图、折线图展示。
- 支持与历史数据对比。

责任边界：

- 数据由甲方提供。
- 涉及查看数据由甲方提供接口。
- 乙方负责呈现和实时提醒。

#### 8.3.2 路径优化建议

需求：

- 基于 AGV 历史行驶数据分析路径拥堵点。
- 示例：某段路径日均拥堵次数 ≥3 次。
- 输出优化建议，如调整路径走向、增设避让点位。
- 支持通过仿真验证优化效果，如拥堵次数减少比例。

责任边界同上。

## 9. 数字孪生监控与管理核心功能需求

### 9.1 全局监控与精准定位

#### 9.1.1 全局状态看板

需求：

- 展示项目整体运行数据。
- 包括立体库货位利用率，已占用货位 / 总货位。
- 包括堆垛机运行率，运行设备数 / 总设备数。
- 包括 AGV 任务完成率。
- 包括故障设备数量。
- 数据实时更新。
- 支持点击看板指标查看明细。
- 示例：点击“货位利用率”跳转至货位分布视图。

#### 9.1.2 精准定位查询

需求：

- 支持按设备编号查询，如堆垛机 / AGV。
- 支持按货位编号查询。
- 支持按货物 SKU 查询。
- 输入 AGV 编号如“AGV-05”，系统可自动聚焦至该 AGV 的数字模型。
- 显示设备实时状态、当前任务。
- 输入货位编号如“R03-L05-C12”，可高亮该货位并显示存放货物信息。

### 9.2 数据统计与报表输出

需求：

- 支持用户配置报表模板。
- 报表内容包括：
  - 堆垛机作业量统计，按日 / 设备。
  - AGV 效率分析，按周 / 类型。
  - 货位周转率，按月 / SKU。
  - 故障统计，按故障类型 / 月份。
- 报表可设置自动生成周期。
- 示例：每日 8 点生成日报。
- 支持邮件自动推送给指定接收人。

责任边界：

- 数据由甲方通过接口提供。
- 乙方负责展示、报表生成和提醒。

### 9.3 远程控制与权限管理

#### 9.3.1 远程控制功能

- 远程控制需要管理员权限。
- 控制指令需要记录操作日志。
- 操作日志包括操作人、时间、指令内容。
- 指令下发延迟要求 ≤2 秒。

#### 9.3.2 分级权限管理

- 至少设置 3 级权限：管理员、操作员、查看员。

权限说明：

- 查看员：只读权限，包括设备状态查看、历史数据查询。

责任边界：

- 若涉及权限，权限数据由甲方通过接口传给客户端。
- 乙方根据权限展示和隐藏相关功能。

## 10. 数据交互与集成技术要求

### 10.1 与物理设备的通信集成

需求：

- 数据采集集成成功率 ≥99.9%。
- 通信中断时，系统需缓存未上传数据。
- 缓存容量需支持至少 1 小时数据量。
- 通信恢复后自动补传。
- 不得丢失数据。

### 10.2 与 WMS 系统对接

需求：

- 从 WMS 获取入库订单。
- 入库订单字段包括货物 SKU、数量、目标货位。
- 从 WMS 获取出库订单。
- 出库订单字段包括货位编号、货物信息、出库优先级。
- 从 WMS 获取货位管理规则，如先进先出。
- 向 WMS 推送设备作业完成状态，如入库完成、出库准确。
- 从 WMS 获取货位实时状态，如空 / 满。
- 故障影响订单预警数据延迟 ≤3 秒。

责任边界：

- 所有数据由甲方通过接口提供或推送。
- 乙方负责展示、实时提醒和必要的接口对接。

### 10.3 与 MES 系统对接

需求：

- 从 MES 获取生产计划。
- 生产计划字段包括物料需求时间、配送点位。
- 向 MES 推送 AGV 物料配送进度，如已送达生产线 A 区。
- 向 MES 推送堆垛机物料出库状态。
- 支持 MES 触发紧急任务，如生产线缺料时优先调度 AGV 配送。

### 10.4 与 ERP 系统对接

需求：

- 向 ERP 同步库存数据，包括每日库存总量、SKU 库存明细。
- 向 ERP 同步设备运行效率数据，如月度设备利用率。
- 向 ERP 同步故障成本数据，如月度故障维修费用。
- 同步频率可配置。
- 默认：每小时 1 次库存快照，每月 1 次效率报表。

## 11. 模型仿真与 WCS 实时对接补充需求

### 11.1 模型仿真定义

本项目中的模型仿真至少分为两类：

#### 11.1.1 实时数据驱动仿真

由 WCS、WMS、MES 或设备系统推送实时数据，数字孪生系统根据数据驱动模型运动和状态变化。

示例：

- WCS 推送 AGV 坐标，AGV 模型移动到对应位置。
- WCS 推送堆垛机 X/Y/Z 坐标，堆垛机模型移动和升降。
- WCS 推送提升机载货台高度，提升机模型更新载货台位置。
- WCS 推送输送线任务状态，输送线显示运行或停止。
- WCS 推送异常，数字孪生模型高亮并弹出告警。

#### 11.1.2 业务任务仿真

当真实设备不动作或处于业务演示模式时，系统根据上游分段任务、路线、目标点位和设备类型，模拟设备运动过程。

示例：

- 按 WMS / WCS 下发的分段任务模拟提升机、输送线、堆垛机连续运行。
- 模拟 AGV 任务分配和路线行驶。
- 模拟仓库订单压力下的设备负载和产能。

### 11.2 与 WCS 系统实时对接

需求：

- WCS 作为设备运行状态、任务状态、坐标状态和异常状态的重要实时数据源。
- 数字孪生系统需要支持与 WCS 采集数据进行对接。
- WCS 实时数据应驱动三维模型状态变化。
- WCS 任务数据应驱动模型仿真流程。

WCS 对接数据建议包括：

- 设备编号。
- 设备类型。
- 设备坐标。
- 运行状态。
- 当前任务号。
- 当前任务阶段。
- 起点。
- 终点。
- 目标货位。
- 托盘号。
- 物料编码。
- 物料名称。
- 数量。
- 电量。
- 速度。
- 故障代码。
- 故障描述。
- 时间戳。

### 11.3 WCS 实时对接接口方式

建议接口方式：

- HTTP Web API，用于任务、状态、查询类接口。
- WebSocket，用于设备实时状态、坐标、告警推送。
- 后续可按现场系统能力增加 TCP、MQ、OPC UA 或其他协议适配。

### 11.4 实时对接性能要求

建议指标：

- 设备状态变化到数字孪生模型更新延迟 ≤2 秒。
- 堆垛机状态、故障信息同步延迟 ≤500ms。
- AGV 坐标采集频率 ≥2 次 / 秒。
- 堆垛机位置信息采集频率 ≥1 次 / 秒。
- 设备远程控制指令下发延迟 ≤2 秒。
- WMS 货位状态数据延迟 ≤3 秒。

## 12. CAD 渲染与 GLB 渲染补充需求

### 12.1 CAD 规划建筑物渲染

需求：

- 系统应支持通过 CAD 资料生成三维可视化场景。
- CAD 渲染内容包括建筑物、厂房、楼层、区域、货架区、地堆区、输送线规划区域、AGV 路径规划区域等。
- CAD 所涉及区域应被转换为可在 Web 端加载和浏览的三维资产。

建议技术路线：

```text
CAD / STEP / IFC / 规划模型
→ 后端或离线工具链转换
→ 生成 GLB 或 3D Tiles
→ 后端生成 manifest
→ 前端 Three.js 加载
```

对于厂区、建筑、楼层等静态大场景，优先使用 3D Tiles。

### 12.2 GLB 模型渲染

需求：

- 系统应支持渲染 GLB 模型。
- GLB 用于设备模型，包括 AGV、堆垛机、提升机、输送线关键段、机械手等。
- GLB 设备模型应支持对象树、点击选中、局部高亮、可动部件绑定和任务动画。

GLB 设备模型要求：

- 保留关键节点。
- 保留可动部件层级。
- 支持可动部件绑定。
- 支持业务名称映射。
- 支持设备状态高亮。
- 支持异常高亮和告警 callout。

### 12.3 CAD 与 GLB 的分工

| 类型 | 推荐格式 | 主要用途 |
|---|---|---|
| 建筑物 / 厂房 / 楼层 / 地面 | 3D Tiles | 静态大场景流式加载 |
| 固定货架 / 管廊 / 固定结构 | 3D Tiles 或低精 GLB | 静态展示 |
| AGV / 堆垛机 / 提升机 / 输送线关键段 | GLB | 可动、可点击、可高亮、可仿真 |
| 业务 Overlay / 告警 / 标签 / 路线 | 前端数据层 | 实时叠加展示 |

## 13. 性能与安全要求整理

### 13.1 系统性能

#### 13.1.1 实时性

- 物理设备状态变化至数字孪生模型更新延迟 ≤2 秒。
- 用户操作响应时间 ≤1 秒。
- 用户操作包括视图切换、设备选中、报表生成等。

#### 13.1.3 多客户端部署

- 支持多个客户端同时部署。
- 支持在多个位置部署客户端同时展示。

### 13.2 安全与权限

- 支持远程控制权限管理。
- 支持操作日志记录。
- 支持分级权限：管理员、操作员、查看员。
- 涉及权限数据时，可由甲方通过接口传给客户端，乙方根据权限显示或隐藏功能。

## 16. 待确认问题清单

1. CAD 文件类型范围：DWG、DXF、STEP、IFC、RVT、其他格式是否都需要支持。
2. CAD 转换由乙方实现、第三方工具实现，还是甲方提供可用模型。
3. WCS 是否提供 WebSocket 实时推送，还是只提供 HTTP 接口。
4. WCS 任务数据是否包含分段任务号、起点、终点、目标设备和时间戳。
5. AGV 坐标系、堆垛机坐标系、提升机坐标系是否有统一坐标标准。
6. 设备远程控制是否仅展示按钮，还是系统需要真正下发控制指令。
7. 调度仿真、路径优化、产能仿真是否需要乙方实现算法，还是只展示甲方提供的数据。
8. 权限体系是否由甲方系统提供，乙方仅展示隐藏功能，还是乙方需独立实现权限。
9. 报表是否需要乙方生成，还是上游系统提供统计数据。
10. 大屏是否需要触摸交互，现场电视是否支持触摸。
11. 虚拟摄像头是否仅为软件视角，还是需要与真实摄像头视频流联动。
12. 性能指标中 16 台堆垛机 + 120 台 AGV 是否为必须验收场景。
13. 路径颜色中“黄色路径”的业务含义需确认，截图备注提示可能不是临时调整路径。

## 17. 结论

本次补充明确了 IDTS 数字孪生项目不仅需要模型资产管理和三维展示，还需要覆盖：

- CAD 规划建筑物渲染。
- GLB 设备模型渲染。
- 模型仿真。
- 仿真与 WCS 实时数据对接。
- AGV、堆垛机、提升机、输送线、立体库和地堆的业务场景展示。
- WMS / MES / ERP 等上游系统数据对接。
- 大屏交互、物资查询、告警、报表、权限和远程控制。

该补充应作为 Word 方案编写的业务输入，不建议直接全部并入 MVP 一期开发。开发阶段仍应遵循现有 MVP 任务拆分，先完成模型资产、GLB 加载、可动部件配置和后端 manifest 闭环，再逐步接入 WCS、仿真、告警、报表和上游系统集成。
