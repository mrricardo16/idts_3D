# POC-3DT-01 测试计划

> 状态：**Ready for review**。通过审核和单独执行授权前不得运行 POC。

## 1. 数据与许可门禁

阶段 A：公开、许可明确的小型 Tileset + 当前 GLB 或 fallback。阶段 B：获授权代表性中大型 Tileset + 当前提升机 GLB。每个样本必须记录来源 URL、许可证、再分发限制、授权人、数据大小和是否可提交仓库；未授权真实数据禁止使用。

## 2. 用例矩阵

| 类别 | Mandatory | 用例与证据 |
|---|---:|---|
| 功能 | 是 | Tiles 与 GLB 同场展示；加载状态、失败状态、GLB 保持可用 |
| 坐标 | 是 | 原点、单位、轴向、比例、三个标定点、偏差记录 |
| 交互 | 是 | Camera、Controls、GLB Raycaster 拾取/高亮不回归 |
| 动画 | 是 | Tiles 存在时 GLB worldZ 动画正常 |
| 错误 | 是 | Tiles URL/请求/解析失败时纯 GLB 回退 |
| 生命周期 | 是 | 取消请求、重复进出、场景切换、销毁与 WebGL 释放 |
| 性能 | 是 | 首屏、可操作时间、FPS、内存、请求数按统一方法记录 |
| 可选扩展 | 否 | Tiles 自身选择、额外缓存或标注策略；不影响 Go 基础判断 |

## 3. 执行与记录

1. 固定浏览器、分辨率、硬件、网络、冷/暖缓存和数据版本。
2. 先跑阶段 A，再确认阶段 B 数据许可和坐标资料。
3. 每项用例记录结果、原始证据位置、异常、责任人和复现步骤。
4. 不以 build 成功、单次截图或主观流畅感替代性能和生命周期证据。

## 4. 三态判定

- Go：所有 Mandatory 通过且证据可复现。
- Conditional Go：Mandatory 中存在经用户批准的受控限制，报告明确责任人、期限、缓解与 MVP-10A 条件。
- No-Go：关键共显、坐标可校准、GLB 交互/动画、回退、生命周期或证据不可复现。

Codex 记录执行和证据，用户或项目负责人作最终批准。
