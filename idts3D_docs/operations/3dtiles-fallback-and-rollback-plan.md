# 3D Tiles 回退与回滚方案

> 状态：**Draft for review**。本方案不执行入口开关、依赖回滚或数据操作。

## 1. 基本原则

3D Tiles 是可隔离的静态底座层，GLB 动态设备必须保留受控可用路径。回退状态必须可见、可诊断，不能将 Tiles 失败伪装为完整场景成功。

## 2. 情形与处理

| 情形 | 即时行为 | 回退 / 回滚边界 |
|---|---|---|
| Tiles 加载、请求或解析失败 | 标记 TilesLayer 失败、保留 DeviceLayer | 关闭 TilesLayer，进入纯 GLB fallback |
| 契约不兼容 | 拒绝不兼容 baseLayers 数据并记录版本 | 回退到上一已审核契约 / 入口关闭 |
| 依赖升级异常 | 停止正式入口发布 | 回滚到经验证依赖版本；不改 GLB 配置 |
| 坐标严重偏差 | 禁止作为正式混合场景发布 | 使用受控 GLB-only 入口，记录校准问题 |
| 数据配置错误 | 隔离对应静态资源引用 | 回滚 scene resource 配置；POC 无数据库回滚 |
| 性能或资源泄漏 | 禁用 TilesLayer 并保留证据 | 回退生产入口，安排独立修复任务 |

## 3. 入口与数据

POC 使用独立入口，不影响 TwinDemo。正式入口关闭、契约版本回退、静态资源配置回退和依赖回退均须由 MVP-10A 的实施方案明确授权。真实数据、生产配置和数据库回滚不在本轮执行范围。

## 4. 验证

回退验证必须证明：GLB 加载、对象交互、worldZ、Monitor / Edit 边界和 fallback 提示可用；页面退出没有残留动画循环或资源引用。
