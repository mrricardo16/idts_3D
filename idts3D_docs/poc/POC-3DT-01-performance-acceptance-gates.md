# POC-3DT-01 性能验收门槛候选

> 状态：候选，等待用户审批。本文不修改现有 `平均 FPS >= 45` 的暂定正式门槛，也不改变 `POC-3DT-01: Fail`。

## 固定测量口径

- 本机 headed Chrome，`1920 × 1080`，deviceScaleFactor `1`，`factory-exterior` 固定相机。
- 每轮 Tiles/GLB ready 后预热 5 秒，再连续 rAF 采样 30 秒；每场景 3 轮。
- 正式 FPS 轮关闭 Trace、视频、连续截图、故障注入、生命周期切换、开发者工具性能录制和请求拦截。
- 网络以 CDP `Network.loadingFinished.encodedDataLength` 为主；暖缓存 3 轮复用同一 BrowserContext，不重启、不清缓存。
- 同时保留开发服务器和 production preview 两类结果；性能结论以已确认正式托管环境的复验为准。

## 通过前置条件

1. 生产静态托管层、版本发布和缓存头责任人已确认。
2. 优化资产使用新文件名与 SHA-256，不覆盖原始 `lifter.glb`。
3. 对合并/实例化/拆分的每个对象都有 `sourceObjectId` 映射，运动件不在静态优化集合中。
4. 所有数值按上方固定口径执行；不得通过缩小视口、隐藏真实 GLB、缩短采样、过滤低帧或降低门槛获取结果。

在用户明确批准前，上述任何候选都不得用于把 POC 状态改为 Pass 或解锁 MVP-10A。
