# 仓库文本与产物治理策略

> 状态：ARCH-02 建立。本文只规定后续治理基线，不对历史文件执行批量编码、换行或格式化重写。

## 1. 文本基线

- 新建或修改的文本文件遵循根目录 `.editorconfig`：UTF-8、默认 LF、文件末尾换行和无意义行尾空白。
- Markdown 允许保留表达硬换行的行尾空格；不得用格式化工具破坏其语义。
- C#、项目文件和 XML 默认使用 4 个空格；前端文本、JSON 与 YAML 默认使用 2 个空格。此规则不引入命名、Analyzer 或格式化器策略。
- PowerShell、CMD 和 BAT 在 Windows 工作区使用 CRLF；Shell 脚本保持 LF。
- 已有 UTF-8 BOM、混合换行、尾随空白或末尾换行问题必须逐文件评估并登记独立治理任务，ARCH-02 不批量修复。

## 2. Git 文本与二进制边界

- `.gitattributes` 是文本与二进制属性的唯一基线。常见源码、配置、契约和脚本文本明确声明换行策略。
- 图片、Office/PDF 文档、压缩包、可执行物、符号文件、字体、GLB/GLTF 和其他二进制扩展名使用 `binary`，不得进行文本转换。
- 不得在业务任务中运行 `git add --renormalize .`，也不得因新增属性文件批量转换历史文件。

## 3. 构建产物与本地文件

- `bin`、`obj`、`dist`、依赖缓存、日志、覆盖率、测试结果、临时文件、数据库导出、上传目录和转换中间产物不得默认入库；新增此类输出前必须先检查 `.gitignore`。
- 示例环境文件、正式静态资源和经审查的项目文档可以入库。机密、连接串和个人机器配置不在本策略中处理，仍由 ARCH-04 治理。
- 不得以 `*.glb`、`debug/**` 或 `reports/**` 的全局忽略规则替代人工判断。

## 4. debug、reports 与历史资料

- 可复现问题、架构验收所需的小型 debug 证据，以及经过人工筛选、长期需要保留的稳定报告，可以入库。
- 带时间戳、本机信息、性能抓取、转换中间文件和可重复生成的报告默认不入库；若需例外，提交前须记录用途、可再生性、是否包含本机路径或机密、保留期限和文件大小。
- 当前已跟踪的 `idts3D_ui/debug/model-structure.csv`、`idts3D_ui/debug/model-structure.json` 与 `idts3D_ui/reports/**` 保留为历史调试或验收证据。本任务不删除、移动或取消跟踪它们；后续新增同类文件仍须逐项人工审核。

## 5. 正式模型与生成模型

- 正式运行所需的静态模型、manifest、配置和图片可以入库；临时转换输出、源 CAD 文件和大体积导出文件不应默认入库。
- GLB 必须作为二进制处理，但正式 GLB 原则上不能只因扩展名被全局忽略。
- 当前 `.gitignore` 的 `idts3D_ui/public/models/**/*.glb` 规则仍保持不变，因此以下本地文件继续被忽略且不属于 ARCH-02 提交范围：

  | 相对路径 | 大小 | 当前引用与性质 | 可再生性 |
  |---|---:|---|---|
  | `idts3D_ui/public/models/lifter.glb` | 293,192,660 bytes | 被 `public/model-configs/lifter.json`、manifest 和前端加载默认路径 `/models/lifter.glb` 引用，是正式运行资产候选。 | 需要已获授权的源模型和可复现转换参数；当前无法据此策略确认。 |
  | `idts3D_ui/public/models/lifter/lifter.high.glb` | 293,192,660 bytes | 被 manifest 和 LOD 配置引用，是高精运行资产候选。 | 现有 LOD 报告称其由 source 复制生成，但正式来源与长期交付方式仍待确认。 |

- 两个文件均由同一 `.gitignore` 规则命中。ARCH-02 不使用 `git add -f`、`.git/info/exclude` 或任何资产移动、删除、重命名、修改来规避该冲突。
- ARCH-02A 将单独确认正式与临时模型目录、来源与授权、大小、Git LFS 需求、部署是否依赖仓库模型、下载脚本或外部制品存储、精确忽略例外规则，以及现有本地 GLB 是否应正式提交。

## 6. 历史文件处理

发现历史编码、BOM、换行或空白问题时，先记录具体文件与功能影响，再以独立 ARCH 任务处理。除本任务新建或允许修改的治理文件外，不得通过仓库级格式化、批量换行转换或批量移除 BOM 处理历史文件。
