# 3D 性能预算

> 状态：**Draft for review**。没有已实测数据；所有阈值在审核前均为 POC 暂定基线。

## 1. 分层预算

| 层级 | 用途 | 状态 |
|---|---|---|
| POC 暂定基线 | 判断同场展示、交互、动画、资源释放是否可重复测量 | 待用户确认，不等于生产阈值 |
| 正式 MVP 验收 | MVP-10A 后的具体环境与回归门槛 | 需 POC 和设计审核输入 |
| 生产阈值 | 现场硬件、数据规模、可用性和长期稳定性门槛 | 不属于当前 MVP，待现场资料确认 |

仓库尚未提供经批准的硬件基线、浏览器矩阵、分辨率、网络条件、Tileset 规模或最终阈值；不得伪造 FPS、内存、首屏或请求数。

## 2. POC 必测指标

- 首屏时间和可操作时间：从导航开始到 Tiles 与 GLB 的状态达到指定可用定义。
- FPS：固定采样窗口记录平均值、低分位与观察时间，采样方法 TBD。
- 内存：浏览器可获得的堆/渲染资源指标及采集限制。
- 网络：Tileset 根文件、子资源和 GLB 请求数、失败数、传输量。
- 场景复杂度：Tileset 大小、瓦片数量、GLB 大小、设备数量、分辨率。
- 生命周期：重复进出后对象/资源计数或可观察内存趋势。

## 3. 测试方法

固定环境并同时记录冷/暖缓存。阶段 A 与 B 分开报告，重复次数、采样工具、原始导出路径和异常均写入结果模板。只报告实际观测值和其条件；没有数据即填 TBD。

## 4. 判定

性能结果与功能、坐标、交互和生命周期共同决定 Go / Conditional Go / No-Go。单项看似流畅、构建成功或局部截图均不能替代整体结论。
