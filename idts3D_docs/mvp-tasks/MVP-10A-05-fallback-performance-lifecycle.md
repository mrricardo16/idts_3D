# MVP-10A-05：回退、性能与生命周期

> 状态：**Blocked**；依赖 MVP-10A-04。

## 1. 任务状态
Blocked，未获用户单独授权不得执行验证或发布判断。
## 2. 任务目标
以批准的预算和真实证据验证故障隔离、GLB-only 回退、性能与资源生命周期，并输出 MVP-10A 总体验收报告。

## 6. 输入

## 7. 输出 / 交付物

## 8. 允许修改范围

## 22. 验收标准

## 23. 回归测试

## 24. 失败停止条件

## 25. 风险

## 26. 回滚方案

## 27. 证据与报告

## 31. 逐故障回退矩阵

下表预先定义执行方法；“结论”填写实际执行结果，不得以未执行替代 Pass。所有情况均要求保留 GLB DeviceLayer、显示可识别的 Tiles fallback 状态，并保存浏览器控制台、Network HAR、截图/录屏和时间戳。

| 故障 | 注入方式 | 检测方式 | 用户可见状态 | 即时处理 | 回退路径 | 恢复条件 | 结论 |
|---|---|---|---|---|---|---|---|
| Tiles URL 失败 | 冻结测试配置指向受控 404 URL | 请求状态/loader error | “静态底座不可用，GLB-only” | 取消该层请求 | 禁用 TilesLayer | 恢复批准 URL 并重新加载成功 | 执行时填写 |
| 根 tileset 失败 | 受控根文件 404 或不可读副本 | 根请求失败/loader error | 同上，标明根资源失败 | 停止解析 | GLB-only | 根文件可读且重新加载 | 执行时填写 |
| 子瓦片失败 | POC 批准样本的受控子资源失败 | 子请求失败、TilesLayer 错误事件 | 部分静态底座失败提示 | 停止失败分支并记录范围 | 卸载 TilesLayer（或冻结策略允许的安全降级） | 子资源恢复且完整性复核通过 | 执行时填写 |
| 解析失败 | POC 批准的无效内容副本 | 解析异常/控制台错误 | “静态资源解析失败” | 中止解析任务 | GLB-only | 内容校验通过并重新加载 | 执行时填写 |
| 坐标错误 | 使用冻结报告登记的受控错误 transform | 三点误差超过冻结阈值 | “坐标校验失败，未发布混合场景” | 禁止发布/隐藏 TilesLayer | GLB-only | 三点在批准阈值内可复现 | 执行时填写 |
| 依赖异常 | 测试中使 Tiles 初始化适配器返回异常 | 初始化异常/错误边界 | “Tiles 依赖初始化失败” | 阻止 TilesLayer 初始化 | GLB-only | 锁定获批依赖版本并重测 | 执行时填写 |

## 32. Pass / Conditional Pass / Fail 判定与解锁

| 维度 | Pass | Conditional Pass | Fail |
|---|---|---|---|
| 功能 | baseLayers/devices 分层、错误隔离均通过 | 非阻断问题有负责人、期限和绕行方案 | 任一 Mandatory 功能失败 |
| GLB 回归 | Object Tree、拾取、worldZ、告警、Edit/Monitor 均通过 | 仅非关键展示问题且 GLB 行为未改变 | 任一 GLB 行为回归 |
| 坐标 | 三点达到冻结阈值并可复现 | 现场绝对容差未定但显式变换稳定复现 | 无法稳定复现或需人工试调 |
| 性能 | 达到批准阶段 A/B 预算 | 仅批准的非阻断偏差，有整改期限 | 未达 Mandatory 阈值或无原始数据 |
| 内存 / 生命周期 | 无重复 RAF/监听，资源趋势符合预算 | 仅可解释且有关闭计划的非阻断观察项 | 泄漏、陈旧请求写入或释放失败 |
| fallback | 十类故障均可见、GLB-only 可用 | 少量非阻断文案/诊断改进待办 | 任一故障破坏 GLB 或无回退 |
| API / 数据库 | 版本、兼容、真实 PostgreSQL/Swagger 验证通过 | 环境限制已获用户书面批准且无契约风险 | 契约不兼容、配置不可回滚或真实验证失败 |
| 证据完整度 | 原始指标、HAR、日志、截图/录屏完整可复核 | 非阻断补件有负责人/期限 | 缺少 Mandatory 证据 |

结论规则：任一 Mandatory Fail 即为 **Fail**；仅存在已登记、有人负责、具关闭期限和证据计划的非阻断问题，才可为 **Conditional Pass**；全部 Mandatory 通过且证据完整才为 **Pass**。Pass 才允许进入 MVP-11～16；Conditional Pass 仅在用户批准、条件/负责人/期限/关闭证据齐全后进入；Fail 必须回退 GLB-only、禁止进入 MVP-11～16，并创建独立修复任务。
