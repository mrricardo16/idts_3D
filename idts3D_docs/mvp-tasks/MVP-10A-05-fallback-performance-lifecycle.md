# MVP-10A-05：回退、性能与生命周期

> 状态：**Blocked**；依赖 MVP-10A-04。

## 1. 任务状态
Blocked，未获用户单独授权不得执行验证或发布判断。
## 2. 任务目标
以批准的预算和真实证据验证故障隔离、GLB-only 回退、性能与资源生命周期，并输出 MVP-10A 总体验收报告。
## 3. 背景与问题
设计已规定 Tiles 可隔离、GLB 必须可用和资源释放清单，但没有实测结果、缓存策略或生产阈值事实。
## 4. 前置条件
10A-04 已独立验收；可用的获授权测试数据、环境、性能预算和回退/生命周期设计齐备。
## 5. 解锁条件
用户确认测试环境、数据许可、采样方案和失败注入授权；未确认时不填写性能结论。
## 6. 输入
10A-04 发布候选、`3d-performance-budget.md`、生命周期/回退规范、POC 证据、纯 GLB 对照和获授权环境信息。
## 7. 输出 / 交付物
回退矩阵、原始性能证据、释放/重复进出记录、总体验收报告和“是否允许进入 MVP-11～16”的用户待决建议。
## 8. 允许修改范围
测试、观测、错误状态、生命周期和回退开关的最小实现及文档；精确文件由 10A-04 实际结构确认。
## 9. 禁止修改范围
不伪造性能/浏览器结论，不放宽预算，不删除可观察错误，不扩展 CAD/IFC/生产切片或无关业务功能。
## 10. 现有文件
`hybrid-scene-resource-lifecycle.md`、`3d-performance-budget.md`、`3dtiles-fallback-and-rollback-plan.md`、TilesLayer/DeviceLayer、TwinScene、ResourceDisposer、InteractionManager。
## 11. 计划新增文件
候选为受控失败注入测试、性能采样/报告和资源计数测试；不预设生产监控或新依赖。
## 12. 前端影响
同步验证请求取消、场景切换、页面卸载、RAF、事件、Raycaster、Geometry/Material/Texture/RenderTarget、Tiles renderer 缓存/重试释放；GLB fallback 状态对用户可见。
## 13. 后端影响
原则上不改业务逻辑；若 API 契约/配置错误需诊断，仅按 10A-04 已定义接口验证，不新增接口掩盖问题。
## 14. 数据库影响
不新增 schema；验证静态资源配置错误时按 10A-04 批准的配置回滚，不在本卡臆造表操作。
## 15. API / DTO / TypeScript 契约
验证不兼容响应、错误可见状态与回退，不改字段；发现契约缺陷回到 10A-01/04 处理。
## 16. 前后端一对一映射
`baseLayers` 错误→TilesLayer 状态/日志/GLB 回退；`devices` 错误→既有 GLB fallback；API 不兼容→拒绝数据与版本回退。不得将错误归入无关层。
## 17. 执行步骤
1. 固定环境；2. 纯 GLB 对照；3. 冷暖缓存各 3 轮；4. 注入全部失败类别；5. 采集性能/网络/内存；6. 连续进出与切换；7. 回退演练；8. 编写总报告。
## 18. 数据准备
记录硬件、浏览器、分辨率、缩放、网络、样本、许可证和缓存状态；阶段 A/B 使用获授权数据。
## 19. 构建命令
使用 10A-04 的已锁定构建/测试命令；性能执行不替代构建，未获授权不得安装分析依赖。
## 20. 自动化测试
覆盖取消、卸载、错误隔离、重复循环/监听、资源释放和契约不兼容；记录无法自动化的 WebGL/性能观测理由。
## 21. 手工验证
逐一验证 URL、根 tileset、子瓦片、解析、坐标、依赖、性能、泄漏、API 不兼容、数据库配置错误；记录检测、用户状态、即时处理、回退、日志和恢复条件。
## 22. 验收标准
按批准预算记录 FPS、1% Low、首屏/可操作时间、内存、网络和重复进入退出；无虚构结果；每个失败均有可观察 GLB-only 路径与证据。
## 23. 回归测试
纯 GLB、GLB Object Tree/拾取/worldZ/告警、monitor/edit、10A-03 坐标、10A-04 Manifest 兼容均须通过。
## 24. 失败停止条件
关键回退不可用、坐标不可复现、泄漏/重复循环、未处理异常、性能未达已批准阈值或证据缺失即停止并不得进入 MVP-11～16。
## 25. 风险
环境不可比、缓存误判、异步资源残留、错误被吞没和以一次关闭替代生命周期验证。
## 26. 回滚方案
关闭 TilesLayer/新入口并恢复 GLB-only；按 10A-04 既定次序回退契约/配置/数据库，保留 HAR、日志和指标。
## 27. 证据与报告
总报告含硬件/浏览器、环境、对照、原始 FPS/1% Low/内存/网络、失败注入、十次进出、回退、TBD、结论和用户批准记录。
## 28. 完成定义
MVP-10A 总体验收报告可供用户审核；只有用户明确批准后才可进入相应 MVP-11～16。
## 29. 下一任务入口
用户审核总报告后，按依赖分别授权 MVP-11～MVP-16；未通过则回到责任子卡。
