# 1. 项目背景与建设目标

IDTS 数字孪生系统拟从当前 WebGL / Three.js 技术预研 Demo 升级为正式工业数字孪生平台，服务于智能仓储、自动化物流和厂区级三维监控场景。系统目标不是单纯展示三维模型，而是围绕厂区空间、业务设备、库存货位、任务流程、告警事件和上游业务数据建立可视化、可追踪、可扩展的数字孪生系统。

- 实现厂区、建筑、楼层、区域、货架、地堆和输送线等空间对象的三维可视化。
- 实现 AGV、堆垛机、提升机、输送线关键段等业务设备的 GLB 渲染、对象选择和状态映射。
- 通过 WCS 或现场系统提供的状态、坐标、任务和告警数据驱动模型变化。
- 支持 AGV、堆垛机、提升机和输送线的运行过程、任务流程和异常状态可视化。
- 支持货位、物资、托盘、任务、告警、报表、大屏和权限等业务联动展示。
- 为后续 WMS、MES、ERP 集成、报表统计和生产化治理预留清晰边界。

项目建设应坚持分阶段落地：先完成模型资产与三维展示基础，再接入实时数据和任务过程，最后扩展报表、大屏、权限、远程控制和生产化运维能力。

# 3. 总体技术架构

推荐架构采用 Vue 3 + TypeScript + Three.js 作为前端三维与管理界面基础，采用 GLB 设备层承载动态业务设备，采用 3D Tiles 静态厂区层承载厂区、建筑、楼层、固定货架和地面等大体量静态对象。后端固定采用 ASP.NET Core Web API / .NET 8 / EF Core 8，并通过 Worker 资产处理服务、数据库、对象存储和接口适配层支撑正式项目落地。

```text
浏览器客户端
  ├─ Vue 3 管理界面
  ├─ Three.js 场景引擎
  ├─ 3D Tiles 静态厂区层
  ├─ GLB 业务设备层
  ├─ Overlay 告警 / 任务 / 状态层
  └─ RuntimeStateAdapter 实时状态适配

IDTS 后端接口（ASP.NET Core Web API / .NET 8）
  ├─ 模型资产 API
  ├─ 场景 / 设备实例 API
  ├─ 可动部件 / Motion Target API
  ├─ WCS / WMS / MES / ERP 接口适配
  └─ 查询、告警、报表、权限与审计接口

资产处理与数据层
  ├─ Worker 资产处理服务
  ├─ CAD / GLB / 3D Tiles 转换工具链
  ├─ 数据库（EF Core 8 管理元数据）
  └─ 对象存储（source / high / medium / low / tiles / manifest）
```

- Three.js 保留为主三维引擎，负责 GLB、设备交互、任务动画、告警浮层和自定义视角。
- 3D Tiles 用于厂区、建筑、楼层、固定货架、地面等静态大场景，解决大体量场景的流式加载问题。
- GLB 用于提升机、堆垛机、AGV、输送线关键段等业务设备，保留对象树、可动部件、材质和业务绑定能力。
- 可动部件、任务动画、异常高亮和业务绑定不依赖 3D Tiles 内部节点，而由 GLB 设备层和业务配置驱动。
- 后端缓存派生模型资产和元数据，不缓存浏览器 GPU 渲染状态。
- 模型转换、LOD、切片、manifest、object-tree、model-stats 等长耗时任务由后端 Worker 或离线工具链处理。

# 4. 三维模型与场景建设方案

## 4.1 CAD 规划建筑物渲染

系统应支持将项目 CAD 规划中的厂区、建筑、楼层、区域、货架区、地堆区、输送线规划区域和 AGV 路径展示区域转换为 Web 可浏览的三维资产。CAD、BIM、IFC、STEP、DWG 等源文件不适合作为浏览器长期运行时加载格式，应由后端或离线工具链转换为 GLB 或 3D Tiles。

- 大型静态场景优先转换为 3D Tiles，以支持空间层级、分区加载和后续大场景生产化。
- 小型或低精度静态模型可采用 GLB，但需要控制文件体积、面数、材质和贴图预算。
- 静态场景应具有版本管理、发布、回滚、校准和审计能力。
- 厂区坐标系、楼层高度、设备摆放坐标和业务路径坐标必须统一。
- 浏览器端只负责加载和展示，不承担复杂 CAD 转换、切片和资产压缩。

## 4.2 GLB 设备模型渲染

提升机、堆垛机、AGV、输送线关键段和机械手等业务设备建议使用独立 GLB 模型接入。GLB 设备模型需要保留对象树、mesh、材质、可动部件层级和可识别业务命名，以支持点击选中、对象详情、父子级查看、异常高亮、任务动画和后续可动部件配置。

- 关键业务对象应保留稳定名称，如设备根节点、固定框架、可动平台、货叉、AGV 主体、输送线段等。
- 模型中应保留业务名称、partCode、运动轴、上下限和目标点位配置所需的可匹配对象。
- 设备 GLB 与厂区底座通过 position、rotation、scale 和统一坐标基准对齐。
- 当前 Demo 中 CAD 转换得到的自动节点名可用于调试，但不宜作为正式长期业务绑定。
- 异常状态、选中状态和业务状态应以可恢复的材质或高亮层实现，避免永久污染原始模型材质。

## 4.3 模型资产管理

正式项目需要从静态文件管理升级为模型资产管理体系，覆盖上传、去重、转换、派生、索引、发布和审计全过程。

| 能力 | 说明 | 实施建议 |
| --- | --- | --- |
| 模型上传 | 支持 GLB 设备模型和后续 CAD / IFC / STEP 等源文件上传。 | MVP 先以 GLB 上传为主，CAD 转换后续阶段进入。 |
| hash 去重 | 对源文件计算 hash，避免重复转换和重复存储。 | 后端统一计算并维护 sourceFileHash。 |
| 版本管理 | 区分 Draft、Ready、Published、Archived、Failed、Invalid 等状态。 | 监控模式只加载 Published 版本。 |
| manifest 加载 | 前端不直接拼接文件路径，而消费后端 manifest。 | manifest 描述 source/high/medium/low/tiles 和 transform。 |
| object-tree 索引 | 保存模型对象树、对象路径、父子级、包围盒和业务绑定索引。 | 用于点击、配置、绑定失效检测和调试。 |
| model-stats | 统计文件大小、mesh、material、texture、vertex、triangle、draw call 估算等。 | 作为发布前性能门禁依据。 |
| 派生资产 | 生成 source / high / medium / low / tiles / thumbnail 等资产。 | 不同视角和场景按需加载不同精度。 |
| 发布 / 回滚 / 审计 | 发布前检查绑定、性能预算、manifest 和转换状态。 | 所有发布、回滚和危险操作记录审计。 |

# 5. 模型仿真方案

模型仿真不是单纯播放固定动画，也不是由数字孪生系统自行计算调度策略，而是由业务数据驱动的运行过程呈现。系统应支持实时数据驱动仿真和业务任务仿真两类模式：实时模式由 WCS / 现场系统推送状态、坐标、任务和告警数据；业务仿真模式根据上游系统分段任务、路线结果和目标点位进行可视化演示。

| 仿真对象 | 仿真内容 | 数据依赖 | 边界说明 |
| --- | --- | --- | --- |
| AGV | 行驶轨迹、方向、速度、任务状态、取货、卸货、充电、故障。 | AGV 坐标、速度、电量、任务、故障数据。 | 路径结果、调度结果和冲突处理结果由第三方调度系统 / WCS / 上游系统提供，本系统负责展示。 |
| 堆垛机 | X/Y/Z 移动、货叉伸缩、取货、放货、返回待机位、异常报警。 | 堆垛机坐标、货叉状态、任务、货位和故障数据。 | 作业路径结果和产能分析结果由上游系统提供，本系统负责三维呈现和回放。 |
| 提升机 | 载货台升降、运行状态、载货有无、任务和异常。 | 提升机高度、任务、传感器或电机状态。 | 运动轴可采用 worldZ 或配置化轴。 |
| 输送线 | 线段运行、停止、堵塞、故障、货物经过过程。 | 输送线段状态、货物位置、异常码。 | 需明确线段编码和交接点。 |
| 货位与物资 | 货位空满、锁定、异常、物资在库、入库、出库、搬运。 | WMS / WCS / 库存和任务数据。 | 库存准确性依赖上游数据。 |

- 实时模式下，设备模型应根据 WCS / 现场系统坐标和状态变化实时更新。
- 业务仿真模式下，系统可根据分段任务展示设备连续运行过程，但任务分配、路径规划、冲突规避和调度计算结果由第三方调度系统 / WCS / 上游系统提供。
- 对于上游系统只提供离散状态的情况，前端可进行插值展示，但不能替代真实定位数据。
- 仿真结果应服务于展示、联动、提醒、回放和复盘；产能数据展示、第三方路径优化结果展示和效率指标呈现均以外部系统提供的数据为依据。

## 5.1 调度与算法边界

数字孪生系统不承担 AGV 调度算法，不承担路径规划算法，不承担冲突规避算法，不承担任务分配算法，也不承担产能优化算法。第三方调度系统、WCS 或其他上游系统负责完成调度计算、路径计算、冲突处理、任务分配和产能优化，并向数字孪生系统推送计算结果。

数字孪生系统负责接收调度结果、路径点、任务状态、设备状态、异常状态和效率指标，并进行三维可视化呈现、状态映射、轨迹展示、回放和提醒。如项目要求数字孪生系统独立承担调度、路径规划、冲突规避、任务分配或产能优化算法，应另立专项评估，不属于当前方案默认建设范围。

# 6. WCS 实时数据对接方案

WCS 是设备状态、任务状态、位置数据和告警数据的重要来源。数字孪生系统应通过 WebAPI、WebSocket、SignalR、HTTP polling 等方式与 WCS 对接。初期可采用 HTTP polling 或 WebAPI 快速闭环；后续实时状态、坐标和告警建议采用 WebSocket / SignalR 推送。

```text
WCS / WMS / MES / ERP
→ IDTS 后端接口
→ 数据清洗与状态映射
→ 前端 RuntimeStateAdapter
→ Three.js 场景实时更新
```

| WCS 提供数据 | 用途 | 数字孪生系统处理 |
| --- | --- | --- |
| 设备编号、设备类型 | 匹配设备实例和模型资产。 | 映射到场景中的设备 GLB 或设备图标。 |
| 设备运行状态 | 显示运行、待机、充电、故障、维护等状态。 | 更新颜色、标签、详情面板和告警状态。 |
| AGV 坐标、速度、电量 | 驱动 AGV 位置、方向、速度和低电量提示。 | 移动 AGV 模型，显示轨迹、电量和充电区占用。 |
| 堆垛机 X/Y/Z 坐标 | 驱动堆垛机横移、升降和货叉动作。 | 更新模型位置、运动部件和作业轨迹。 |
| 提升机高度 | 驱动载货台升降。 | 按配置化运动轴或 worldZ 更新可动部件。 |
| 输送线状态 | 显示线段运行、停止、堵塞和故障。 | 更新输送线段状态和异常提示。 |
| 当前任务号、起点、终点 | 展示任务进度和路径。 | 绘制任务轨迹线，更新任务详情。 |

数字孪生系统负责接收数据、映射到设备实例、更新模型位置和状态、高亮异常部件、展示任务过程、展示告警提醒并提供查询追踪。上游系统负责提供准确、及时、可解释的业务数据；本系统不应在未约定范围内自行推断业务调度结果。

# 7. 设备级业务功能设计

## 7.1 AGV 数字孪生

- 展示 AGV 路线、实时坐标、行驶轨迹、方向、速度和当前任务。
- 展示任务号、起点、终点、优先级、任务状态和执行进度。
- 展示取货、运输、卸货过程和当前搬运物料信息。
- 展示电量、充电状态、充电区空闲 / 占用和低电量预警。
- AGV 故障时高亮模型或图标，展示故障类型、可能原因和处理建议。
- 路径拥堵、避让、调度建议等结果由第三方调度系统或 WCS 提供，本系统负责展示和追踪；如需本系统独立计算，应另立专项，不属于当前方案范围。

## 7.2 堆垛机数字孪生

- 展示堆垛机运行、待机、故障、维护等运行状态。
- 展示 X 轴货架列、Y 轴货架层、Z 轴升降高度等位置信息，位置精度建议按需求达到 ±10mm。
- 展示货叉伸、缩、举升等动作，以及取货、放货、返回待机位等作业流程。
- 展示当前任务、目标货位、托盘、SKU、数量、入库时间、所属订单等信息。
- 故障或货位异常时立即高亮并提示，支持弹窗、告警列表和声光提醒。
- 作业路径结果展示、效率指标展示和产能分析结果呈现可作为后续阶段，计算结果由第三方调度系统 / WCS / 上游系统提供，本系统负责三维呈现、轨迹回放和提醒。

## 7.3 提升机数字孪生

- 展示提升机运行 / 待机 / 故障状态、升降高度、载货台位置和当前任务。
- 载货台应作为可动部件配置，支持 worldZ 或配置化运动轴。
- 通过 WCS 或设备接口提供的高度数据驱动升降动作。
- 展示载货台有货 / 无货、传感器异常、电机异常和任务状态。
- 与输送线、堆垛机、AGV 的接驳点需要在场景中明确标识。

## 7.4 输送线数字孪生

- 展示输送线运行、停止、故障、堵塞和停线状态。
- 按输送线段显示状态，支持线段级启停、异常和货物经过过程。
- 展示当前输送物料信息和异常原因。
- 与堆垛机、提升机、AGV 的交接点应可视化，便于跟踪任务流转。

## 7.5 立体库、货位和地堆

- 展示货位空 / 满 / 锁定 / 异常状态。
- 点击货位查看库位编码、托盘编码、SKU、名称、规格、数量、入库时间和所属订单。
- 空料架点击显示托盘编码；满料架点击显示物料详情。
- 支持按货位、托盘、SKU、物料名称和规格型号查询，查询后高亮定位。
- 支持立体库和地堆区域展示，货位状态由 WMS / WCS / 库存接口提供。

## 8.5 数据边界说明

- 上游系统提供业务数据、任务数据、库存数据、权限数据、设备控制结果和调度计算结果。
- 数字孪生系统负责展示、映射、提醒、查询、定位、回放和仿真呈现。
- 调度算法、路径优化、冲突规避、任务分配和产能优化不属于本系统默认建设范围；第三方调度系统 / WCS / 上游系统提供计算结果，本系统负责接收和展示。
- 远程控制涉及安全风险，应明确控制接口、权限、二次确认、审计和责任边界。

# 9. 交互与展示功能

- 支持缩放、旋转、拖拽视角和多视角切换。
- 支持全局鸟瞰视角、堆垛机特写视角、AGV 路径视角和设备跟随视角。
- 支持虚拟摄像头、巡检路线、自定义区域轮播和固定大屏视角。

- 支持点击设备查看详情、点击货位查看物资详情、查询物资后高亮定位。
- 支持告警 callout、弹窗提示、异常设备高亮、任务轨迹线展示和历史告警入口。
- 触摸缩放和旋转是否启用取决于现场大屏硬件；不支持触摸时应支持鼠标和键盘操作。

# 11. 远程控制与权限管理

远程控制属于高风险能力，应作为后续增强能力逐步进入。初期建议先实现监控展示、告警提醒和权限显示边界；真正下发控制指令前，需要确认控制接口、权限体系、现场安全机制、二次确认和审计要求。

| 权限类型 | 建议权限范围 | 控制要求 |
| --- | --- | --- |
| 管理员 | 系统配置、设备控制、权限分配、报表管理、发布回滚。 | 危险操作二次确认，全部写入审计日志。 |
| 操作员 | 设备状态查看、任务跟踪、故障报警接收、部分操作如报表导出或视图配置。 | 是否允许任务取消、手动调度需甲方确认。 |
| 查看员 | 只读查看设备状态、历史数据、报表和三维场景。 | 隐藏控制按钮和配置入口。 |

- 远程控制能力包括堆垛机紧急停止、AGV 任务取消、AGV 手动调度等。
- 控制指令需记录操作人、时间、设备、指令内容、执行结果和失败原因。
- 控制指令下发延迟需求为 ≤2 秒，但必须结合现场控制链路实测。
- 涉及权限数据时，可由甲方通过接口传给客户端，数字孪生系统根据权限显示或隐藏功能。

# 12. 性能与安全要求

| 类别 | 指标 | 说明 |
| --- | --- | --- |
| 实时性 | 物理设备状态变化到数字孪生模型更新延迟 ≤2 秒。 | 设备状态、告警和位置更新应在可接受延迟内完成。 |
| 操作响应 | 用户操作响应时间 ≤1 秒。 | 包括视图切换、设备选中、查询和报表触发等常用操作。 |
| 并发渲染 | 16 台堆垛机 + 120 台 AGV 同时运行时，系统帧率 ≥25fps，无明显卡顿。 | 需结合模型规模、客户端硬件和网络条件实测。 |
| 稳定运行 | 连续运行无故障时间 ≥720 小时 / 月。 | 约 30 天，属于生产化运行指标。 |
| 恢复能力 | 软件故障恢复时间 ≤30 分钟。 | 需要部署、日志、监控和运维流程配合。 |
| 数据完整性 | 硬件故障恢复后数据补传完整性 ≥99.9%。 | 依赖缓存、补传和上游数据链路。 |
| 多客户端 | 支持多个客户端同时部署和展示。 | 需考虑数据推送、会话和性能隔离。 |
| 采集成功率 | 数据采集成功率 ≥99.9%。 | 接口、网络和现场系统共同保障。 |
| 断线补传 | 通信中断时缓存至少 1 小时数据，恢复后自动补传。 | 缓存位置和责任边界需确认。 |
| 安全审计 | 权限控制和操作审计。 | 远程控制、配置修改和发布回滚必须审计。 |

# 15. 风险与边界

| 风险 / 边界 | 说明 | 控制建议 |
| --- | --- | --- |
| CAD 转换复杂 | 不同 CAD / BIM / IFC / STEP / DWG / RVT 格式兼容性差。 | 先明确源文件格式和样例，建立工具链验证流程。 |
| 浏览器转换能力有限 | 浏览器不适合长期承担复杂 CAD 转换、压缩、切片和 LOD。 | 由后端 Worker 或离线工具链处理资产。 |
| 大场景 GLB 风险 | 全厂区使用巨大 GLB 会导致首屏慢、内存高、交互卡顿。 | 静态大场景优先 3D Tiles，设备使用 GLB。 |
| 坐标系不一致 | CAD、3D Tiles、GLB、WCS 坐标可能存在方向、单位和原点差异。 | 建立统一坐标基准和转换规则，早期做对齐验证。 |
| 仿真准确性依赖数据 | 设备运动与业务状态依赖 WCS / 上游系统数据质量。 | 明确字段、频率、时间戳和异常处理机制。 |
| 离散状态插值 | 如果上游只提供离散状态，前端只能做插值展示。 | 在接口协议中明确位置频率和插值策略。 |
| 算法责任边界 | 路径优化、调度算法、产能分析若要求独立计算，工作量显著扩大。 | 默认由第三方调度系统 / WCS / 上游系统提供计算结果，本系统负责呈现；若本系统计算需另立专项。 |
| 调度算法责任误解 | 若将调度、路径规划、冲突规避误认为数字孪生系统职责，会显著扩大项目范围。 | 明确由第三方调度 / WCS / 上游系统提供结果，数字孪生系统只负责呈现和追踪。 |
| 远程控制安全 | 紧急停止、任务取消、手动调度涉及现场安全。 | 权限、二次确认、审计、接口确认和现场联调不可缺少。 |
| 性能验收不确定 | 性能受模型规模、硬件、网络、浏览器和现场部署影响。 | 建立模型预算、性能门禁和现场实测验收标准。 |

# 17. 结论

本方案建议采用 Three.js + GLB + 3D Tiles + .NET 8 后端的总体路线：Three.js 继续作为主三维引擎；CAD 建筑和厂区静态场景优先转换为 3D Tiles；提升机、堆垛机、AGV、输送线关键段等业务设备使用 GLB；模型仿真由 WCS / 上游系统数据驱动；数字孪生系统负责三维展示、状态映射、任务可视化、告警提醒、查询定位和报表展示。

项目应分阶段落地，先完成基础三维展示与模型资产管理，再接入 WCS 实时数据和设备状态映射，随后扩展模型仿真、大屏、报表、权限、审计和生产化治理。对于依赖上游系统的数据、调度结果、路径结果和产能分析结果，应在接口协议、责任边界和验收标准中提前明确。
